'[笔记]血泪教训集'

这里记录了我和我的伙伴们的错题，作为警示，以防再错。

自己的血泪

关于 long long

  • 爆long long最好的办法是使用long double.(对于整型)

(括号内的类型表示存储类型,实际上并不能像这样进行强制类型转换)

xxxxxxxxxx
 (long long)  pow ( 2 , 64 ) == 0
(long double) pow ( 2 , 64 ) == 18446744073709551616.000000

按范围给算法

  • 如果不能保证正解的完全正确性,在小数据时使用暴力也是不错的.(至少不会一错全错)

像这样:

xxxxxxxxxx
if ( n <= 1000 ) 
{
       brute () ; 
}
else if ( n <= 10000 ) 
{
       solution () ; 
}

临界表存图与空间

  • 无向图存边,开双倍空间!(天知道我为这错误调试了多久)

对于无向图临界表存边来说,下面这操作很显然这样会数组越界(一条无向边 等于 两条互通的有向边)

xxxxxxxxxx
//MAXM 是图的边数
node e[ MAXM ] ; 
//接着在主函数里
while ( m -- )
{
    u = read () ; 
    v = read () ; 
    addedge ( u , v ) ; 
    addedge ( v , u ) ;
}

那么显然,上面的做法等待它的是 RE 和 WA ,所以应该这么干:

xxxxxxxxxx
node e[ 2 * MAXM ] ;

BFS 的 flag 标记

  • 用 BFS 水图论题的分还是可以的，但一定要记得对 flag 标记进行检查和更新！

比如下面这个图上 BFS 就会超时：

xxxxxxxxxx
deque < int > que ;
bool flag [ MAXN ] ; 
inline bool bfs ( int source , int goal ) 
{ 
    memset ( flag , 0 , sizeof ( flag ) ) ; 
    que . clear ( ) ; 
    que . push_back ( source ) ; 
    flag [ source ] = true ; 
    while ( ! que.empty ( ) )
    {
        source = que . front ( ) ;
        que . pop_front ( ) ; 
        for ( int i = list [ source ] ; i ; i = e [ i ] . last )
        {
            const int v = e [ i ] . to ; 
            if ( ! flag [ v ] )
            {
                if ( v == goal )
                {
                    return true ; 
                }
                que . push_back ( v ) ;
            }
        }
    }
    return false ; 
}

所以要这么改：

xxxxxxxxxx
deque < int > que ;
bool flag [ MAXN ] ; 
inline bool bfs ( int source , int goal ) 
{ 
    memset ( flag , 0 , sizeof ( flag ) ) ; 
    que . clear ( ) ; 
    que . push_back ( source ) ; 
    flag [ source ] = true ; 
    while ( ! que.empty ( ) )
    {
        source = que . front ( ) ;
        que . pop_front ( ) ; 
        for ( int i = list [ source ] ; i ; i = e [ i ] . last )
        {
            const int v = e [ i ] . to ; 
            if ( ! flag [ v ] )
            {
                flag [ v ] = true ; //起死回生，妙手回春
                if ( v == goal )
                {
                    return true ; 
                }
                que . push_back ( v ) ;
            }
        }
    }
    return false ; 
}

欲用矩阵·先判重边

  • 虽然 邻接表 和 链式前向星 非常香，但有时 邻接矩阵 可能会更好。不过一定要注意重边的判断！

例如这样就会被重边卡：

xxxxxxxxxx
bool e [ MAXN ] [ MAXN ] ; 
e [ u ] [ v ] = true ; //加边
e [ u ] [ v ] = false ; //删边

所以要这么干：

xxxxxxxxxx
int e [ MAXN ] [ MAXN ] ; 
e [ u ] [ v ] ++ ; //加边
e [ u ] [ v ] -- ; //删边

Dinic 的异或问题

  • 解决网络最大流的最好办法当然就是是 Dinic 了（ 因为 Ford-Fulkerson 和 Edmond-Karp 几乎没有实战价值，HLPP 没有可写性 ），但是对于邻接表存储的反向边的处理时的^一定要注意！
xxxxxxxxxx
e [ i ] . cap -= d ; 
e [ i ^ 1 ] . cap += d ;

对，这没问题，问题在于存边的edgecnt。 如果是这样的话：

xxxxxxxxxx
int edgecnt = 0 ；

就会出错。因此，从 1 开始是个不错的选择：

xxxxxxxxxx
int edgecnt = 1 ;

网络流/二分图的两个点集

  • 对于有些 网络流/二分图 的题目，题意有这样的：

我有两个点集， 每个点集最多有N个点。

但要注意了，不是每个这种题意的题目都这么显然， 所以一定要注意 MAXN 的大小， 一般建议这么做：

xxxxxxxxxx
// 定义 MAXN 
# define MAXN 100
// 使用 MAXN 
struct edge 
{
    int to , last , cap ; 
} e [ MAXN * 2 ] ;

或者干脆：

xxxxxxxxxx
# define MAXN 100*2

别人的教训

空格邪教(来自Jesse26)

  • 打空格固然是个不错的习惯,但是对于 "" 里的内容来说可就不太好了.

这样是对的:

xxxxxxxxxx
freopen ( "code.in" , "r" , stdin ) ;

但是这样就不对了:

xxxxxxxxxx
freopen ( "code.in" , " r" , stdin ) ;

万恶的/(来自Eqvpkbz)

  • 在做除法运算时一定要注意是否有/0的运算,nan和inf可不是闹着玩的.

如果有下面这些情况,那就尴尬了.

xxxxxxxxxx
long double a , b ; 
//经历一系列运算后,a==1,b==0
printf ( "%Lf" , a / b ) ; 
//我们便得到了 inf
xxxxxxxxxx
long double a , b ; 
//又经历一系列运算后,a==0,b==0
printf ( "%Lf" , a / b ) ; 
//我们又有了 nan
  • 同时浮点运算也有锅

比如这样:

xxxxxxxxxx
long double a ; 
//经历一系列运算后,a == -0.000000
//对,很让人疑惑,但终端里就是"-0.000000"
//我也没办法......

静态调试万般好(来自zmy123456)

  • 当 VScode 或 Dev-cpp 的调试坏了的时候，就可以在程序里输出变量进行查看。

像这样：

xxxxxxxxxx
printf ( "---DEBUG---\n" ) ; 
printf ( "x=%d\ty=%d\n" , galaxy [ i ] . x , galaxy [ i ] . y) ; 
printf ( "size=%d\n" , galaxy [ i ] . size) ; 
printf ( "ulcnt=%d\n" , galaxy [ i ] . ulcnt ) ; 
printf ( "urcnt=%d\n" , galaxy [ i ] . urcnt ) ; 
printf ( "planet=" ) ;
for ( int j = 0 ; j < galaxy [ i ] . size ; j ++ )
{
    cout << " " << galaxy [ i ] . planet [ j ]  ; 
}
printf ( "\n----END----\n" ) ;
  • 但是一定要记得删除或注释静态调试语句！

不然就会：

Wrong answer on line 1 , read "------" , expect "62137" .
